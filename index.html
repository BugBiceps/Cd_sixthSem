Unit 1

Introduction to Compiling & Lexical Analysis

Compiler structure, models of compilation, translation phases, and lexical tools. A steady, moderate presence across every paper — mostly definitional and diagram-based.

14–28
marks per paper on average
~20%
of the total printed paper
~3
sub-questions per paper
Translators & basic concepts
  • What is a translator? Compare compiler and interpreter in terms of translation method, memory requirement and speed.
  • Illustrate the concepts of compiler and interpreter.
Compiler types, architecture & models
  • What is a cross compiler? Give an example.
  • List and explain the advantages and disadvantages of frontend and backend model of compiler.
  • Discuss the analysis and synthesis model of compilation. Explain each phase using the input: a = (b + c)*(b + c)*2
  • Explain the various compiler construction tools.
Phases of a compiler
  • Explain in detail the various phases of compilers with an example. repeated
  • Explain the various phases of a compiler with a neat sketch. repeated
  • Draw the structure of a compiler and describe each phase. Mention the output of id1 = id2 + id3 * 50 at each phase.
  • Solve a := b + c * 4 through the different phases of the compiler.
Lexical analysis & input buffering
  • Explain the following term with an example: lexical analysis.
  • What are the issues to consider in the design of a lexical analyzer? Why is buffering used in lexical analysis?
  • Discuss the functionality of pre-processing and input buffering.
  • Write a short note on: specification and recognition of tokens.
The LEX tool
  • Explain the LEX tool in brief.
  • Design a LEX program that recognizes the tokens of a C language and returns the token found.
Focus: "phases of a compiler" is the single most repeated prompt in this unit — be ready to draw the pipeline and trace one expression through every stage from memory.
Unit 2

Syntax Analysis & Syntax Directed Translation

The heaviest unit in the syllabus: grammars, parsing algorithms (top-down and bottom-up), and attribute grammars. December 2024 alone carried 60 marks across its sub-questions.

28–36
marks per paper (peak: 60)
~30–35%
of the total printed paper
4–7
sub-questions per paper
Grammar basics, ambiguity & transformations
  • What is a context-free grammar? Explain its closure properties.
  • Describe: (i) ambiguity, (ii) left recursion.
  • What is left factoring in grammar? Explain with an example.
  • Eliminate left recursion for: S → Aa | b, S → Ac | Sd | ε
  • Prove that if L is a recursive language, then L is also a recursive language.
  • Find First and Follow sets for: S→aAB/bA/ε, A→aAb/ε, B→bB/ε
  • For S→a/^(T), T→T,S/S — find leftmost derivation, rightmost derivation and parse tree for (((a,a),^(a)),a).
Parsing concepts & top-down parsing
  • Differentiate between top-down and bottom-up parsing, with examples. repeated
  • What is a parser? Explain backtracking and non-backtracking parsers with their types.
  • What is LL(1) grammar? Give its properties.
  • Construct a recursive-descent parser with backtracking for: S → aSbS | bSaS | ε
Bottom-up parsing: SLR & LALR
  • What are the various conflicts that occur during shift-reduce parsing?
  • Write a short note on: SLR parser.
  • Compute LR(0) items and construct an SLR parsing table for: S → L=R | R, L → *R | id, R → L
  • Construct an SLR parsing table for: E → E+T, T → TF/F, F → F*/a/b
  • Construct a LALR table for E→E+T/T, T→T*F/F, F→(E)/id, and verify id + id * id.
  • Construct a LALR parsing table for: E → E+T/T, T → T*F/F, F → (E)/id
Syntax directed translation & attributes
  • Define with examples: (i) synthesized attributes, (ii) annotated parse tree, (iii) dependency graph.
  • Differentiate synthesized and inherited attributes, with suitable examples.
  • Differentiate S-attributed and L-attributed definitions, with an example.
  • Write a short note on: L-attributed definition.
  • What is S-attributed and L-attributed SDT? Write SDT rules for: S→S*A, S→A, A→A+B, B→(S), A→B, B→id
  • Explain an SDT to convert infix expressions to postfix.
  • Given translation rules with E as the start symbol, compute E.value for 2 « 3 & 5 # 6 & 4.
Focus: you must be able to construct SLR and LALR tables from a blank grammar, and write or evaluate SDT rules — this half of the syllabus alone carries roughly 50% of your attemptable score.
Unit 3

Type Checking & Run Time Environment

How a compiler verifies data types and manages memory during execution. Highly irregular: absent in May 2022 and December 2024, but worth up to 28 marks in June 2025.

~13
marks per paper (0–28 range)
~12%
of the total printed paper
2–4
sub-questions when tested
Type checking & equivalence
  • Explain in detail the role of a type checker in a compiler.
  • Explain in brief the equivalence of type expressions.
  • Explain different polymorphic functions with suitable examples.
Run time environment & storage allocation
  • Discuss the various storage allocation strategies in detail. May '24 & Jun '25
  • How is stack storage allocation different from heap allocation? Describe both, with merits and demerits.
  • What is an activation record? Explain the model of an activation record.
Short notes
  • Write a short note on: dynamic storage allocation.
  • Write a short note on: error detection and recovery.
Focus: unpredictable appearance, but when it shows up it leans on storage allocation strategies — know stack vs heap and the activation record model cold.
Unit 4

Code Generation

The bridge from front-end analysis to machine code: intermediate code, DAGs, flow graphs and basic blocks. Highly practical and consistently heavy every single year.

20–35
marks per paper (avg ~28)
~25%
of the total printed paper
4–5
sub-questions per paper
Intermediate code & three-address code (3AC)
  • Explain various intermediate code generation techniques in brief.
  • Why are quadruples preferred over triples in an optimizing compiler?
  • Write the 3AC sequence for: P := (X + Y) + (X - C)
  • Generate 3AC for a nested while/if-else block with conditions on a, b, c, d.
  • Generate code for: R = (p + q) - ((r + s) - t)
  • Write a short note on: three address code.
  • Write a short note on: back patching. May '22 & Jun '25
Basic blocks & flow graphs
  • What are the steps for partitioning three-address statements into basic blocks?
  • Define a flow graph. Explain how a program is converted into one.
  • Represent in a flow graph: i=1; sum=0; while(i<=10){ sum+=i; i++; }
  • Construct a flow graph for a given array-indexing program (prod, i, t1..t6).
Directed acyclic graphs (DAG)
  • What is a DAG? Draw the DAG for a = (a*b+c) - (a*b+c)
  • Construct a DAG for: D:=B*C; E:=A+B; B:=B+C; A:=E-D;
  • Construct a DAG for a basic block with T1..T4 assignments.
  • Write an algorithm to construct a DAG from a basic block.
  • What are the applications of a DAG, and how do DAGs help implement transformations on basic blocks?
Register allocation & peephole optimization
  • Discuss register allocation and assignment with examples.
  • Discuss peephole optimization / write a short note on it.
Focus: practice drawing DAGs and flow graphs and writing 3AC by hand — this unit is guaranteed to show up as a diagram or numerical problem, worth 40% of your attemptable score.
Unit 5

Code Optimization

Making generated code faster: basic block optimizations, loop invariant computations, and global data flow analysis. Reliable but moderate, and mostly theoretical.

12–14
marks per paper (peak: 21)
~11–12%
of the total printed paper
1–3
sub-questions per paper
Sources & areas of optimization
  • Explain the different sources of optimization of basic blocks, with a suitable example. highly repeated
  • Explain in detail the principal sources of optimization.
  • What are the three areas of code optimization? Explain each in detail.
Code improving transformations
  • Mention the criteria for code-improving transformations.
  • What is local transformation and global transformation?
  • Explain common sub-expression elimination, copy propagation, and moving loop-invariant computations.
Global data flow analysis & short notes
  • Write a brief note on: global data flow equations.
  • Explain the following term with an example: loop optimization.
  • Write a short note on: common sub-expression elimination.
You've reached the end. Memorize the sources of optimization and the code-improving transformations — that's most of this unit's marks in two topics.
